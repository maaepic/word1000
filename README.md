# Задача 

В docx (word-2007) файле сделать "толстым" (bold) N-ное по порядку слово, начиная с 1.
При этом разметка не должна сломаться.

notes:
 * docx формат является zip-архивом с набором xml файлов внутри,
 * нужный файл называется "document.xml" и лежит в папке word,
 * ...


## Зависимости

Нужен zip и unzip.


## Простейшее решение (шел)
папка `Shell`

Такое себе, но для простых документов вполне рабочее решение. На самом деле просто забавно что awk+sed вполне хватает для решения задачки.
А главное - черновое, простое и быстрое (10 минут на весь "кодинг").

Требует sed, awk.

Использование:  
`./main.sh filename.docx`

На выходе `a.docx`.


## C + mxml library

Рабочее. Разметка сохраняется. Считает слова и не считает словами одинокие пробелы.

Требует mxml (https://github.com/michaelrsweet/mxml).

Использование:  
```shell
cd C
make # (mxml скачается и соберется)
./main.sh filename.docx 13
```

На выходе `a.docx`.

Требует побольше тестирования, может нагенерю примеров.
